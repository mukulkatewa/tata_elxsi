# Phase 4: Live Vehicle Dashboard - Quick Start Guide

## Overview

The AutoForge Live Vehicle Dashboard is a real-time visualization system that demonstrates vehicle health monitoring capabilities through simulated Vehicle Signal Specification (VSS) compliant data. Watch as vehicle signals update in real-time, inject fault scenarios, and monitor alerts as they occur.

## Running the Dashboard

### Prerequisites

Ensure all dependencies are installed:

```bash
pip install -r requirements.txt
```

Required packages: `streamlit`, `plotly`, `pandas`, `numpy`

### Launch Command

From the project root directory, run:

```bash
python run_dashboard.py
```

The dashboard will automatically open in your browser at `http://localhost:8501`

Press `Ctrl+C` in the terminal to stop the dashboard.

## Dashboard Features

### Real-Time Gauges

The dashboard displays six live gauges showing current vehicle status:

- **Speedometer**: Vehicle speed (0-200 km/h)
  - Green: 0-80 km/h (safe city speeds)
  - Yellow: 80-130 km/h (highway speeds)
  - Red: 130-200 km/h (high speed)

- **Tyre Pressure Gauges** (4 tyres: FL, FR, RL, RR): Pressure in kPa
  - Red: Below 180 kPa (low pressure warning)
  - Green: 180-280 kPa (optimal range)
  - Yellow: 280-350 kPa (over-inflated)

- **Battery Gauge**: State of charge (0-100%)
  - Red: 0-20% (low battery)
  - Yellow: 20-40% (moderate)
  - Green: 40-100% (good charge)

### Trend Charts

Four time-series charts show signal history over the last 60 ticks:

1. **Tyre Pressure Trends**: All four tyre pressures with 180 kPa warning threshold
2. **Battery & Range**: Dual-axis chart showing battery percentage and estimated range
3. **Vehicle Speed**: Speed variations over time
4. **Temperature Monitoring**: Motor and coolant temperatures with warning thresholds

### Alert System

Active alerts appear at the top of the dashboard with color-coded severity:

- **WARNING** (yellow): Signal approaching unsafe levels
  - Tyre pressure < 180 kPa
  - Battery < 20%
  - Motor temperature > 100°C
  - Coolant temperature > 95°C

- **CRITICAL** (red): Immediate attention required
  - Tyre pressure < 150 kPa
  - Battery < 10%
  - Motor temperature > 115°C
  - Coolant temperature > 105°C

### Scenario Controls (Sidebar)

Use the sidebar to inject fault scenarios and control simulation speed:

**Available Scenarios:**

1. **Normal**: Baseline driving with no faults - realistic random variations

2. **Tyre Puncture**: Front-left tyre pressure drops from 220 to 80 kPa over 30 ticks
   - Watch the FL gauge turn red
   - Alerts trigger at 180 kPa (WARNING) and 150 kPa (CRITICAL)

3. **Low Battery**: Battery drains 3x faster than normal
   - Accelerated discharge demonstrates battery monitoring
   - Range decreases proportionally

4. **Overheating**: Motor temperature climbs from 80 to 118°C over 20 ticks
   - Temperature gauge shows progressive heating
   - Alerts trigger at 100°C (WARNING) and 115°C (CRITICAL)

5. **Highway Cruise**: Speed maintains steady 120 km/h
   - Simulates cruise control on highway
   - Minimal speed variation (±2 km/h)

6. **City Driving**: Speed oscillates between 0-60 km/h
   - Simulates stop-and-go traffic
   - 15-tick oscillation period

**Controls:**

- **Scenario Selector**: Choose from dropdown menu
- **Apply Button**: Activate the selected scenario
- **Reset Button**: Return to normal operation
- **Tick Speed Slider**: Adjust simulation speed (0.1 - 2.0 seconds per tick)

### Bottom Tabs

Three tabs provide additional information:

1. **Raw Signals**: Complete data table showing all 13 VSS signals with timestamps
2. **Alert Log**: Historical record of all alerts generated during the session
3. **Generated Code**: View code generated by Phase 2 (if available)

## Usage Tips

### Demonstrating Fault Detection

1. Start with **Normal** scenario to establish baseline
2. Apply **Tyre Puncture** to watch pressure drop and alerts trigger
3. Use **Reset** to clear the fault
4. Try **Overheating** to see temperature-based alerts
5. Apply **Low Battery** to demonstrate battery monitoring

### Adjusting Simulation Speed

- Use the **Tick Speed** slider to control update frequency
- Slower speeds (1.5-2.0s) are better for presentations
- Faster speeds (0.1-0.5s) show trends more quickly

### Monitoring Multiple Signals

- Watch how scenarios affect related signals
- Example: Low battery reduces EV range proportionally
- Example: Highway cruise changes gear position and throttle

## Integration with AutoForge Phases

The dashboard integrates with the complete AutoForge ecosystem:

- **Phase 1 (RAG Knowledge Base)**: Provides VSS documentation context
- **Phase 2 (Multi-Agent Codegen)**: Generated services appear in sidebar
- **Phase 3 (Self-Healing Build)**: Ensures generated code quality
- **Phase 4 (Live Dashboard)**: Demonstrates the complete system

The dashboard operates independently and doesn't require other phases to function.

## Troubleshooting

### Dashboard Won't Start

**Error: "streamlit is not installed"**
```bash
pip install -r requirements.txt
```

**Error: "File not found: autoforge/dashboard/app.py"**
- Ensure you're running from the project root directory
- Check that the `autoforge/dashboard/` directory exists

### Browser Doesn't Open Automatically

Manually navigate to: `http://localhost:8501`

### Dashboard Appears Frozen

- Check the terminal for errors
- Refresh the browser page
- Restart the dashboard with `Ctrl+C` and rerun

### Scenarios Not Working

- Ensure you clicked the **Apply** button after selecting a scenario
- Use **Reset** to clear any stuck states
- Check the terminal for error messages
